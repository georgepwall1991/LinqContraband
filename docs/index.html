LinqContraband

An EF Core LINQ performance analyzer and Roslyn analyzer that catches query problems before they ship: N+1 loops, premature materialization, sync-over-async, missing includes, raw SQL risks, tracking mistakes, and more.

Official project

  • Canonical repository: georgepwall1991/LinqContraband
  • Official package: NuGet LinqContraband
  • Maintainer: George Wall

LinqContraband is a NuGet analyzer package. It is not distributed as a standalone ZIP installer or executable.

What It Catches

N+1 Query Patterns

Finds database calls inside loops, missing includes, deep eager loading, and cartesian explosion risks before production latency appears.

Query Translation Traps

Flags local methods, string conversions, untranslatable overloads, nondeterministic pagination, and other LINQ shapes EF Core cannot translate safely.

Raw SQL and Tracking Risks

Catches interpolated raw SQL, constructed SQL strings, unsafe query-filter bypasses, tracking-mode mixups, and silent no-tracking writes.

Install

dotnet add package LinqContraband

The analyzer runs at compile time in Visual Studio, Rider, VS Code, and CI. It adds no runtime dependency to your application.
